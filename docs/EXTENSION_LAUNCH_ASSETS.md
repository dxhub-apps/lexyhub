## Chrome Web Store Review Guidelines

To avoid rejection, ensure:

### Do's âœ…
- Single, focused purpose (keyword research)
- Clear description of functionality
- All permissions necessary and explained
- Privacy policy linked in manifest
- High-quality screenshots showing actual extension
- Responsive to user reviews

### Don'ts âŒ
- Misleading descriptions or screenshots
- Keyword stuffing in description
- Requesting unnecessary permissions
- Scrapers or data extraction tools (we're okay - this is user research)
- Cryptocurrency mining
- Obfuscated code
- Spam or repetitive content

### Common Rejection Reasons
1. **Permissions not justified** - Explain in description why each permission is needed
2. **Unclear functionality** - Show screenshots of actual features
3. **Privacy policy missing** - Must be linked in manifest.json
4. **Trademark issues** - Use "for Etsy" not "Etsy Tool"
5. **Low-quality assets** - Use high-resolution screenshots
